40  /**
41    * <!-- globalinfo-start --> Changes the order of the classes so that the class
42    * values are no longer of in the order specified in the header. The values will
43    * be in the order specified by the user -- it could be either in
44    * ascending/descending order by the class frequency or in random order. Note
45    * that this filter currently does not change the header, only the class values
46    * of the instances, so there is not much point in using it in conjunction with
47    * the FilteredClassifier. The value can also be converted back using
48    * 'originalValue(double value)' procedure.
49    * <p/>
50    * <!-- globalinfo-end -->
51    *
52    * <!-- options-start --> Valid options are:
53    * <p/>
54    *
55    * <pre>
56    * -R <seed>
57    * Specify the seed of randomization
58    * used to randomize the class
59    * order (default: 1)
60    * </pre>
61    *
62    * <pre>
63    * -C <order>
64    * Specify the class order to be
65    * sorted, could be 0: ascending
66    * 1: descending and 2: random.(default: 0)
67    * </pre>
68    *
69    * <!-- options-end -->
70    *
71    * @author Xin Xu (xx5@cs.waikato.ac.nz)
72    * @author Eibe Frank (eibe@cs.waikato.ac.nz)
73    * @version $Revision: 12037 $
74    */
