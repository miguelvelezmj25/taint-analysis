41  /**
42    * This class can generate the properties object that is normally loaded from
43    * the <code>GenericObjectEditor.props</code> file (= PROPERTY_FILE). It takes
44    * the <code>GenericPropertiesCreator.props</code> file as a template to
45    * determine all the derived classes by checking the classes in the given
46    * packages (a file with the same name in your home directory overrides the the
47    * one in the weka/gui directory/package). <br>
48    * E.g. if we want to have all the subclasses of the <code>Classifier</code>
49    * class then we specify the superclass ("weka.classifiers.Classifier") and the
50    * packages where to look for ("weka.classifiers.bayes" etc.):
51    *
52    * <pre>
53    *
54    * weka.classifiers.Classifier=\
55    * weka.classifiers.bayes,\
56    * weka.classifiers.functions,\
57    * weka.classifiers.lazy,\
58    * weka.classifiers.meta,\
59    * weka.classifiers.trees,\
60    * weka.classifiers.rules
61    *
62    * </pre>
63    *
64    * This creates the same list as stored in the
65    * <code>GenericObjectEditor.props</code> file, but it will also add additional
66    * classes, that are not listed in the static list (e.g. a newly developed
67    * Classifier), but still in the classpath. <br>
68    * <br>
69    * For discovering the subclasses the whole classpath is inspected, which means
70    * that you can have several parallel directories with the same package
71    * structure (e.g. a release directory and a developer directory with additional
72    * classes). <br>
73    * <br>
74    * The dynamic discovery can be turned off via the <code>UseDyanmic</code>
75    * property in the props file (values: true|false).
76    *
77    * @see #CREATOR_FILE
78    * @see #PROPERTY_FILE
79    * @see #USE_DYNAMIC
80    * @see GenericObjectEditor
81    * @see weka.core.ClassDiscovery
82    * @author FracPete (fracpete at waikato dot ac dot nz)
83    * @version $Revision: 13477 $
84    */
