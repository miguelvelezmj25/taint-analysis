42  /**
43    * RollbackTracker is used to detect rollback periods in the log that are the
44    * result of HA replica syncups. These rollback periods affect how LNs should
45    * be processed at recovery. Rollbacks differ from aborts in that a rollback
46    * returns a LN to its previous version, whether intra or inter-txnal, while an
47    * abort always returns an LN to its pre-txn version.
48    *
49    * What is a Rollback Period?
50    * --------------------------
51    * The rollback represents the logical truncation of the log. Any transactional
52    * LNs in that rollback period should be undone, even if they are ultimately
53    * part of a committed transaction. See the wiki page on Syncup Recovery for
54    * the full design. See com.sleepycat.je.rep.impl.node.Replay.rollback for the
55    * steps taken at the time of the rollback.
56    *
57    * A RollbackStart record is logged at the start of any rollback, and a
58    * RollbackEnd is logged at the completion of a rollback. RollbackStarts refer
59    * to a matchpoint and the area between the matchpoint and the RollbackStart is
60    * the rollback period.The RollbackTracker peruses RollbackStarts and Ends and
61    * generates a map of the rollback periods.
62    *
63    * RollbackStarts and their starting Matchpoints can be nested or can be
64    * distinct, but several invariants are in place and can be enforced. For
65    * example:
66    *
67    * LSN
68    * ---
69    * 100 txnA commit
70    *
71    * 200 txnB abort
72    * 250 LN for txnC
73    * 300 txnC abort
74    * ..
75    * 400 RollbackStart A (starting matchpoint = 200)
76    * 500 RollbackEnd A
77    * ...
78    * 600 RollbackStart B (starting matchpoint = 200)
79    * 700 RollbackStart C (starting matchpoint = 100)
80    * 800 RollbackEnd C
81    * 900 txnD abort
82    * 1000 RollbackStart D (starting matchpoint = 900)
83    *
84    * This log creates four rollback periods
85    * 1) LSN 100 -> 700 (defined by RollbackStart C). This has two rollback
86    * periods nested within.
87    * 2) LSN 200 -> 400, (defined by RollbackStart A) nested within B
88    * 3) LSN 200 -> 600, (defined by RollbackStart B) nested within C
89    * 4) LSN 1000 -> 900 (defined by RolbackStart D)
90  
91    * - There can be no commits or aborts within a rollback period, because we
92    * shouldn't have executed a soft recovery that undid a commit or abort. in
93    * the rollback period.
94    *
95    * - There can be no LN_TXs between a RollbackStart and its matching
96    * RollbackEnd (should be no LN write operations happening during the syncup.)
97    * However, there might be INs written by a checkpoint, and eviction.
98    *
99    * - The recovery period should never see a RollbackEnd without its matching
100    * RollbackStart record, though it is possible to see a RollbackStart that has
101    * no RollbackEnd.
102    *
103    * - There can never be any overlapping, or intersection of periods, because a
104    * rollback period is supposed to be like a truncation of the log. Since that
105    * log is "gone", a subsequent rollback shouldn't find a matchpoint inside
106    * another rollback period.
107    *
108    * - A child period must be wholly contained between the parent's matchpoint
109    * and RollbackStart. This is simply due to the way rollbacks occur. A parent
110    * rollback has a Matchpoint <= the child's Matchpoint or it wouldn't be
111    * nested. The parent's RollbackStart > the child's RollbackEnd, since the
112    * parent occurs after the child in time.
113    *
114    * The Rollback tracker keeps a list of all the rollback periods. Some are
115    * distinct, some are nested.
116    *
117    * Recovery processing and rollback periods
118    * ----------------------------------------
119    * The actions taken at a rollback may not have been made persistent to the
120    * log, so at recovery, we literally mimic and replay these two steps: (a) make
121    * sure invisible log entries have their invisible bit on and (b) make sure all
122    * INs reflect the correct LNs. All use of the rollback periods and tracker
123    * take place on the backwards scans. The RollbackStart and End entries are
124    * read during the first recovery undo pass When a rollback period is found, a
125    * transaction chain is constructed for each transaction that was active in the
126    * period, to support a repeat of the actions taken originally.
127    *
128    * The first undo pass, for the mapping tree, has to construct a map of
129    * recovery periods. Since the mapping tree only has MapLNs, and we never write
130    * any txnal MapLNs, that first pass does not encounter any txnal LNs. The
131    * next two undo passes consult the rollback period map to determine if an LN
132    * needs to be rolledback, or just treated like other LNs.
133    *
134    * Rollback periods that precede the checkpoint start can be ignored, because
135    * we can be assured that all the INs and LNs modified by that rollback were
136    * made persistent by the checkpoint. Ignoring such periods is required, and
137    * is not just an optimization, because it guarantees that we will not need to
138    * create a transaction chain that needs to traverse the log beyond the first
139    * active lsn. A rollback period precedes the checkpoint if its RollbackEnd is
140    * before the checkpoint start.
141    *
142    * When a rollback period overlaps CkptStart and we recover, we are guaranteed
143    * that the undo passes will process all LNs in the rollback period, because
144    * they are >= to the firstActiveLEnd of the checkpoint.
145    *
146    * The lastActiveLSN for the checkpoint will be <= the LSN of the first LN of
147    * any transaction that is being rolled back at the time of CkptStart, since
148    * these transactions were still active at that time.
149    *
150    * No file containing a transaction rolled back in the recovery interval, or a
151    * file containing the abortLSN of such a transaction, will be deleted by the
152    * cleaner. An active transaction prevents cleaning of its first logged entry
153    * and beyond. The LN of the abortLSN will be locked, which prevents it from
154    * being cleaned.
155    *
156    * All the work lies on the undo side. Recovery redo only needs to ignore
157    * invisible log entries, because we know that the undo pass applied the
158    * invisible bit where needed. Note that the undo pass must be sure to write
159    * the invisible bits after the pass, before redo attempts to read the log.
160    *
161    * Each rollback LN_TX belongs to a single rollback period. When periods are
162    * nested, the LN_TX belongs to the closest rollback period that encompasses
163    * it.
164    * Using the example above,
165    * a LN at lsn 350 belongs to rollback period A
166    * a LN at lsn 550 belongs to rollback period B
167    * a LN at lsn 650 belongs to rollback period C
168    * It uses its rollback period's txn chain to find its previous version.
169    */

318       /**
319        * Flip the invisible bit for each lsn in rollbackLsns. Collect the
320        * corresponding unique set of file numbers and add them to fileNums.
321        */
322       private static void setInvisible(EnvironmentImpl envImpl,
323                                        List<Long> rollbackLsns,
324                                        Set<Long> filesToFsync) {
325           if (rollbackLsns.size() == 0) {
326               return;
327           }
328  
329           /*
330            * Sort so that the entries are made invisible in disk order for better
331            * efficiency.
332            */
333           FileManager fileManager = envImpl.getFileManager();
334  
335           Collections.sort(rollbackLsns);
336           List<Long> perFileLsns = new ArrayList<Long>();
337           long currentFileNum = -1;
338  
339           for (Long lsn : rollbackLsns) {
340  
341               /* See if we have moved to a new file. */
342               if (DbLsn.getFileNumber(lsn) != currentFileNum) {
343                   /*
344                    * We've moved on to a new file. Make the previous set of
345                    * lsns invisible.
346                    */
347                   fileManager.makeInvisible(currentFileNum, perFileLsns);
348  
349                   currentFileNum = DbLsn.getFileNumber(lsn);
350                   filesToFsync.add(currentFileNum);
351  
352                   /* make a new set to house the lsns for the next file. */
353                   perFileLsns = new ArrayList<Long>();
354               }
355               perFileLsns.add(lsn);
356           }
357  
358           /* Take care of the last set. */
359           fileManager.makeInvisible(currentFileNum, perFileLsns);
360       }
361  
362       /*
363        * Flip the invisible bit for the rollback set of lsns, in lsn order.
364        * Fsync the set of files represented in this collection of lsns. Used by
365        * syncup rollback.
366        */
367       public static void makeInvisible(EnvironmentImpl targetEnvImpl,
368                                        List<Long> rollbackLsns) {
369  
370           Set<Long> fsyncFiles = new HashSet<Long>();
371           setInvisible(targetEnvImpl, rollbackLsns, fsyncFiles);
372           targetEnvImpl.getFileManager().force(fsyncFiles);
373       }

378        * for the next round.

382               return;
383           }
384  
385           setInvisible(envImpl,
386                        singlePassInvisibleLsns,
387                        recoveryFilesToSync);
388           singlePassInvisibleLsns = new ArrayList<Long>();
389       }

391       void recoveryEndFsyncInvisible() {
392           if (envImpl.isReadOnly()) {
393               return;
394           }
395  
396           envImpl.getFileManager().force(recoveryFilesToSync);
397       }

399       /**
400        * Count an LN obsolete that is being made invisble by rollback.
401        *
402        * Use inexact counting. Since invisible entries are not processed by the
403        * cleaner, recording the obsolete offset would be a waste of resources.
404        * Since we don't count offsets, we don't need to worry about duplicate

406        *

410        */

412                                  UndoReader undo,
413                                  RecoveryUtilizationTracker tracker) {
414           tracker.countObsoleteUnconditional
415               (undoLsn,
416                null /*type*/,
417                undo.logEntrySize,
418                undo.db.getId(),
419                false /*countExact*/);
420       }

441       /**
442        * A Scanner is to process LNs during a recovery pass. It determines
443        * whether this log entry is within the rollback period, and should be
444        * accordingly undone or ignored. It serves as a sort of cursor or iterator
445        * that works with the rollback tracker.
446        */
447       abstract class Scanner {
448  
449           /*
450            * The target period is the one which houses the LNs that will be
451            * rolled back.
452            */
453           RollbackPeriod target;
454  
455           /**
456            * Return true if this transactional log entry is something that should
457            * be rolled back in this rollback period. The Scanner's position can
458            * be changed by this call. Update the target field if necessary.
459            */
460           abstract boolean positionAndCheck(long lsn, long txnId);
461  
462           /**
463            * Rollback the filereader's current LN_TX. This assumes that the the
464            * caller has ascertained that the LN is contained within this rollback
465            * period.
466            */
467           public void rollback(Long txnId,
468                                LNFileReader reader,
469                                RecoveryUtilizationTracker tracker) {
470  
471               /*
472                * If this period is before the checkpoint start, we need not
473                * repeat the partial rollback.
474                */
475               if (target.beforeCheckpointStart()) {
476                   return;
477               }
478  
479               long undoLsn = reader.getLastLsn();
480  
481               DbTree dbTree = envImpl.getDbTree();
482  
483               UndoReader undo = UndoReader.createForRecovery(reader, dbTree);
484  
485               if (undo == null) { → [ALLOWCREATE]
486                   /* Database of LN has been deleted. [#22052] */
487                   return;
488               }
489          
490               /* Get the TxnChain for this log entry. */
491               TxnChain chain = target.getChain(txnId, undoLsn, envImpl);
492  
493               try {
494                   RevertInfo revertTo = chain.pop();
495  
496                   /*
497                    * When we undo this log entry, we've logically truncated it
498                    * from the log. Remove it from the btree and mark it obsolete.
499                    */
500                   RecoveryManager.rollbackUndo(
501                       envImpl.getLogger(), Level.FINER, new TreeLocation(),
502                       undo.db, undo.logEntry, undoLsn, revertTo);
503  
504                   if (!target.hasRollbackEnd()) { → [ALLOWCREATE]
505  
506                       /*
507                        * We're not positive that the fsync of the invisible log
508                        * entries happened. Make it invisible again.
509                        */
510                       if (!reader.isInvisible()) { → [ALLOWCREATE]
511                           singlePassInvisibleLsns.add(undoLsn);
512                       }
513                   }
514               } finally {
515                   dbTree.releaseDb(undo.db);
516               }
517  
518               countObsolete(undoLsn, undo, tracker);
519           }
520  
521           /* For unit tests */
522           boolean needsRollback() {
523               if (target == null) {
524                   return false;
525               }
526  
527               return !target.beforeCheckpointStart();
528           }
529       }

531       class UnderConstructionScanner extends Scanner {
532  
533           @Override
534           public boolean positionAndCheck(long lsn, long txnId) {
535  
536               if (underConstructionPeriod == null) {
537                   return false;
538               }
539  
540               assert underConstructionPeriod.notInRollbackStartAndEnd
541                   (lsn, txnId) :
542                  underConstructionPeriod.bracketFailure(lsn);
543  
544               target = underConstructionPeriod.getScannerTarget(lsn);
545               if ((target != null) && (target.containsLN(lsn, txnId))) {
546                   return true;
547               }
548  
549               return false;
550           }
551       }
552  
553       /**
554        * In a backward scanner, the currentPeriod field is always pointing to the
555        * period that contains this lsn. If the lsn is not in a period, the
556        * currentPeriod is the period that is just before this lsn. If there is no
557        * period before this lsn, the currentPeriod field is null.
558        */
559       class BackwardScanner extends Scanner {
560           private final Iterator<RollbackPeriod> iter;
561  
562           /*
563            * The current period is the period where the scanner is currently
564            * posed. It is one of the top level periods in the scanner. When
565            * rollback periods are nested, currentPeriod may not equal target.
566            */
567           private RollbackPeriod currentPeriod;
568  
569           BackwardScanner() {
570               this.iter = periodList.iterator();
571               if (iter.hasNext()) {
572                   currentPeriod = iter.next();
573                   currentPeriod.initChildIter();
574               } else {
575                   currentPeriod = null;
576               }
577           }
578  
579           @Override
580           public boolean positionAndCheck(long lsn, long txnId) {
581               if (currentPeriod == null) {
582                   return false;
583               }
584  
585               if (currentPeriod.follows(lsn)) {
586  
587                   /*
588                    * We've passed out of the currentPeriod. Look for a new one
589                    * that might cover this lsn.
590                    */
591                   if (iter.hasNext()) {
592                       currentPeriod = iter.next();
593                       currentPeriod.initChildIter();
594                   } else {
595                       currentPeriod = null;
596                       return false;
597                   }
598               }
599  
600               assert currentPeriod.notInRollbackStartAndEnd(lsn, txnId) :
601                  currentPeriod.bracketFailure(lsn);
602  
603               if (currentPeriod.contains(lsn)) {
604  
605                   /*
606                    * Make the stack of periods point to the one that contains
607                    * this lsn, or which precedes this lsn.
608                    */
609                   currentPeriod.positionChildren(lsn);
610  
611                   /*
612                    * See if any period contains this lsn. There might not be a
613                    * target if the lsn was aborted or committed already at the
614                    * time of rollback.
615                    */
616                   target = currentPeriod.findTarget(lsn, txnId);
617                   return (target != null);
618               }
619  
620               return false;
621           }
622       }

624       /**

704           private boolean calcBeforeCheckpoint(long checkpointStart) {
705               return ((checkpointStart != DbLsn.NULL_LSN) &&
706                       (rollbackEndLSN != DbLsn.NULL_LSN) &&
707                       (DbLsn.compareTo(rollbackEndLSN, checkpointStart) < 0));
708           }

710           /**
711            * A new RollbackEnd has been seen.
712            *
713            * @return true if the RollbackEnd belongs to a period nested within
714            * the current period. Return false if the RollbackEnd belongs to new,
715            * distinct, different period, and the current period is closed.
716            */
717           boolean makeNestedPeriod(RollbackEnd foundRBEnd, long foundLSN) {
718               RollbackPeriod target = getNewPeriodTarget(foundRBEnd, foundLSN);
719               if (target != null) {
720                   target.makeChild(foundRBEnd, foundLSN);
721                   return true;
722               }
723               return false;
724           }
725  
726           /**
727            * A new RollbackStart has been seen.
728            *
729            * @return true if the RollbackStart belongs to a period nested within
730            * the current period, or if it is the current period. Return false if
731            * the RollbackStart belongs to new, distinct, different period, and
732            * this current period is closed.
733            */
734           boolean makeNestedPeriod(RollbackStart foundRBStart, long foundLSN) {
735               RollbackPeriod target = getNewPeriodTarget(foundRBStart, foundLSN);
736               if (target != null) {
737                   if (target.isMatchingRollbackStart(foundLSN)) {
738                       assert target.activeTxnIds == null;
739                       target.activeTxnIds = foundRBStart.getActiveTxnIds();
740                   } else {
741                       target.makeChild(foundRBStart, foundLSN);
742                   }
743  
744                   /*
745                    * Retrun true to let the caller know that it doesn't have to
746                    * make a new Rollback period. Either the RBStart did not
747                    * initiate a new period, or we made a nested child.
748                    */
749                   return true;
750               }
751               /* This period is closed. */
752               return false;
753           }

755           private boolean contained(RollbackEnd foundRBEnd, long foundLSN) {
756  
757               /*
758                * This RollbackEnd must either
759                * 1 - precede this period, in which case this period is closed, or
760                * 2 - be wholly contained within this period.
761                */
762  
763               /* case 1 */
764               if (DbLsn.compareTo(foundLSN, matchpointLSN) < 0) {
765                   /* The found rollback end precedes this period. */
766                   return false;
767               }
768  
769               if (DbLsn.compareTo(foundLSN, rollbackStartLSN) >= 0) {
770                   fail("Should not be two RollbackEnds in a row. " +
771                        "New RollbackEnd at " +
772                        DbLsn.getNoFormatString(foundLSN) +
773                        " " + foundRBEnd);
774               }
775  
776               /*
777                * Check for compliance to the rule that this RollbackEnd does not
778                * intersect this rollback period.
779                */
780               if (!((DbLsn.compareTo(foundRBEnd.getMatchpoint(),
781                                      matchpointLSN) >= 0) &&
782                     (DbLsn.compareTo(foundRBEnd.getRollbackStart(),
783                                      rollbackStartLSN) < 0))) {
784                   fail("RollbackEnd intersects current rollback period " +
785                        foundRBEnd + " at " + DbLsn.getNoFormatString(foundLSN));
786               }
787  
788               /* case 2 */
789               return true;
790           }
791  
792           /**
793            * @return true if the current rollback period is still open
794            */
795           private boolean contained(RollbackStart foundRBStart, long foundLSN) {
796  
797               /*
798                * This RollbackStart must:
799                * 1 - precede the current period, indicating the end of this
800                * period.
801                * 2 - is the rolblack start that belongs to this period.
802                * 3 - be wholly contained within this period.
803                */
804  
805               /* case 1 */
806               if (DbLsn.compareTo(foundLSN, matchpointLSN) < 0) {
807                   /* The found rollback start precedes this period. */
808                   return false;
809               }
810  
811               if (isMatchingRollbackStart(foundLSN)) {
812                   return true;
813               }
814  
815               /* Check for compliance with case 3. */
816               if (!((DbLsn.compareTo(foundRBStart.getMatchpoint(),
817                                      matchpointLSN) >= 0) &&
818                     (DbLsn.compareTo(foundLSN, rollbackStartLSN) < 0))) {
819                   fail("RollbackStart intersects current rollback period " +
820                        foundRBStart + " at " +
821                        DbLsn.getNoFormatString(foundLSN));
822               }
823  
824               /* case 3. */
825               return true;
826           }

828           /**
829            * @return true if this RollbackStart entry is the one that is the

831            */
832           private boolean isMatchingRollbackStart(long foundLSN) {
833               return (DbLsn.compareTo(foundLSN, rollbackStartLSN) == 0);

921           /**

923            * the matchpoint and the RollbackStart. We don't use RollbackEnd,

930                   (DbLsn.compareTo(rollbackStartLSN, lsn) > 0);
931           }

975               if (!TxnManager.isReplicatedTxn(txnId)) {
976                   /* Don't bother checking a non-replicated txn. */
977                   return true;
978               }

981                   return true;
982              
983               return (!((DbLsn.compareTo(rollbackStartLSN, lsn) < 0) &&
984                         (DbLsn.compareTo(rollbackEndLSN, lsn) > 0)));
985           }
