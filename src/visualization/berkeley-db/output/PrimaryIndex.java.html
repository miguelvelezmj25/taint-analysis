56  /**
57    * The primary index for an entity class and its primary key.
58    *
59    * <p>{@code PrimaryIndex} objects are thread-safe. Multiple threads may
60    * safely call the methods of a shared {@code PrimaryIndex} object.</p>
61    *
62    * <p>{@code PrimaryIndex} implements {@link EntityIndex} to map the primary
63    * key type (PK) to the entity type (E).</p>
64    *
65    * <p>The {@link Entity} annotation may be used to define an entity class and
66    * the {@link PrimaryKey} annotation may be used to define a primary key as
67    * shown in the following example.</p>
68    *
69    * <pre class="code">
70    * {@literal @Entity}
71    * class Employee {
72    *
73    * {@literal @PrimaryKey}
74    * long id;
75    *
76    * String name;
77    *
78    * Employee(long id, String name) {
79    * this.id = id;
80    * this.name = name;
81    * }
82    *
83    * private Employee() {} // For bindings
84    * }</pre>
85    *
86    * <p>To obtain the {@code PrimaryIndex} for a given entity class, call {@link
87    * EntityStore#getPrimaryIndex EntityStore.getPrimaryIndex}, passing the
88    * primary key class and the entity class. For example:</p>
89    *
90    * <pre class="code">
91    * EntityStore store = new EntityStore(...);
92    *
93    * {@code PrimaryIndex<Long, Employee>} primaryIndex =
94    * store.getPrimaryIndex(Long.class, Employee.class);</pre>
95    *
96    * <p>Note that {@code Long.class} is passed as the primary key class, but the
97    * primary key field has the primitive type {@code long}. When a primitive
98    * primary key field is used, the corresponding primitive wrapper class is used
99    * to access the primary index. For more information on key field types, see
100    * {@link PrimaryKey}.</p>
101    *
102    * <p>The {@code PrimaryIndex} provides the primary storage and access methods
103    * for the instances of a particular entity class. Entities are inserted and
104    * updated in the {@code PrimaryIndex} by calling a method in the family of
105    * {@link #put} methods. The {@link #put} method will insert the entity if no
106    * entity with the same primary key already exists. If an entity with the same
107    * primary key does exist, it will update the entity and return the existing
108    * (old) entity. For example:</p>
109    *
110    * <pre class="code">
111    * Employee oldEntity;
112    * oldEntity = primaryIndex.put(new Employee(1, "Jane Smith")); // Inserts an entity
113    * assert oldEntity == null;
114    * oldEntity = primaryIndex.put(new Employee(2, "Joan Smith")); // Inserts an entity
115    * assert oldEntity == null;
116    * oldEntity = primaryIndex.put(new Employee(2, "Joan M. Smith")); // Updates an entity
117    * assert oldEntity != null;</pre>
118    *
119    * <p>The {@link #putNoReturn} method can be used to avoid the overhead of
120    * returning the existing entity, when the existing entity is not important to
121    * the application. The return type of {@link #putNoReturn} is void. For
122    * example:</p>
123    *
124    * <pre class="code">
125    * primaryIndex.putNoReturn(new Employee(1, "Jane Smith")); // Inserts an entity
126    * primaryIndex.putNoReturn(new Employee(2, "Joan Smith")); // Inserts an entity
127    * primaryIndex.putNoReturn(new Employee(2, "Joan M. Smith")); // Updates an entity</pre>
128    *
129    * <p>The {@link #putNoOverwrite} method can be used to ensure that an existing
130    * entity is not overwritten. {@link #putNoOverwrite} returns true if the
131    * entity was inserted, or false if an existing entity exists and no action was
132    * taken. For example:</p>
133    *
134    * <pre class="code">
135    * boolean inserted;
136    * inserted = primaryIndex.putNoOverwrite(new Employee(1, "Jane Smith")); // Inserts an entity
137    * assert inserted;
138    * inserted = primaryIndex.putNoOverwrite(new Employee(2, "Joan Smith")); // Inserts an entity
139    * assert inserted;
140    * inserted = primaryIndex.putNoOverwrite(new Employee(2, "Joan M. Smith")); // <strong>No action was taken!</strong>
141    * assert !inserted;</pre>
142    *
143    * <p>Primary key values must be unique, in other words, each instance of a
144    * given entity class must have a distinct primary key value. Rather than
145    * assigning the unique primary key values yourself, a <em>sequence</em> can be
146    * used to assign sequential integer values automatically, starting with the
147    * value 1 (one). A sequence is defined using the {@link PrimaryKey#sequence}
148    * annotation property. For example:</p>
149    *
150    * <pre class="code">
151    * {@literal @Entity}
152    * class Employee {
153    *
154    * {@literal @PrimaryKey(sequence="ID")}
155    * long id;
156    *
157    * String name;
158    *
159    * Employee(String name) {
160    * this.name = name;
161    * }
162    *
163    * private Employee() {} // For bindings
164    * }</pre>
165    *
166    * <p>The name of the sequence used above is "ID". Any name can be used. If
167    * the same sequence name is used in more than one entity class, the sequence
168    * will be shared by those classes, in other words, a single sequence of
169    * integers will be used for all instances of those classes. See {@link
170    * PrimaryKey#sequence} for more information.</p>
171    *
172    * <p>Any method in the family of {@link #put} methods may be used to insert
173    * entities where the primary key is assigned from a sequence. When the {@link
174    * #put} method returns, the primary key field of the entity object will be set
175    * to the assigned key value. For example:</p>
176    *
177    * <pre class="code">
178    * Employee employee;
179    * employee = new Employee("Jane Smith");
180    * primaryIndex.putNoReturn(employee); // Inserts an entity
181    * assert employee.id == 1;
182    * employee = new Employee("Joan Smith");
183    * primaryIndex.putNoReturn(employee); // Inserts an entity
184    * assert employee.id == 2;</pre>
185    *
186    * <p>This begs the question: How do you update an existing entity, without
187    * assigning a new primary key? The answer is that the {@link #put} methods
188    * will only assign a new key from the sequence if the primary key field is
189    * zero or null (for reference types). If an entity with a non-zero and
190    * non-null key field is passed to a {@link #put} method, any existing entity
191    * with that primary key value will be updated. For example:</p>
192    *
193    * <pre class="code">
194    * Employee employee;
195    * employee = new Employee("Jane Smith");
196    * primaryIndex.putNoReturn(employee); // Inserts an entity
197    * assert employee.id == 1;
198    * employee = new Employee("Joan Smith");
199    * primaryIndex.putNoReturn(employee); // Inserts an entity
200    * assert employee.id == 2;
201    * employee.name = "Joan M. Smith";
202    * primaryIndex.putNoReturn(employee); // Updates an existing entity
203    * assert employee.id == 2;</pre>
204    *
205    * <p>Since {@code PrimaryIndex} implements the {@link EntityIndex} interface,
206    * it shares the common index methods for retrieving and deleting entities,
207    * opening cursors and using transactions. See {@link EntityIndex} for more
208    * information on these topics.</p>
209    *
210    * <p>Note that when using an index, keys and values are stored and retrieved
211    * by value not by reference. In other words, if an entity object is stored
212    * and then retrieved, or retrieved twice, each object will be a separate
213    * instance. For example, in the code below the assertion will always
214    * fail.</p>
215    * <pre class="code">
216    * MyKey key = ...;
217    * MyEntity entity1 = new MyEntity(key, ...);
218    * index.put(entity1);
219    * MyEntity entity2 = index.get(key);
220    * assert entity1 == entity2; // always fails!
221    * </pre>
222    *
223    * @author Mark Hayes
224    */
