35  /**
36    * A secondary database handle.
37    *
38    * <p>Secondary databases are opened with {@link
39    * Environment#openSecondaryDatabase Environment.openSecondaryDatabase} and are
40    * always associated with a single primary database. The distinguishing
41    * characteristics of a secondary database are:</p>
42    *
43    * <ul>
44    * <li>Records are automatically added to a secondary database when records are
45    * added, modified and deleted in the primary database. Direct calls to
46    * <code>put()</code> methods on a secondary database are prohibited.</li>
47    * <li>The {@link #delete delete} method of a secondary database will delete
48    * the primary record and as well as all its associated secondary records.</li>
49    * <li>Calls to all <code>get()</code> methods will return the data from the
50    * associated primary database.</li>
51    * <li>Additional <code>get()</code> method signatures are provided to return
52    * the primary key in an additional <code>pKey</code> parameter.</li>
53    * <li>Calls to {@link #openCursor openCursor} will return a {@link
54    * SecondaryCursor}, which itself has <code>get()</code> methods that return
55    * the data of the primary database and additional <code>get()</code> method
56    * signatures for returning the primary key.</li>
57    * </ul>
58    * <p>Before opening or creating a secondary database you must implement
59    * the {@link SecondaryKeyCreator} or {@link SecondaryMultiKeyCreator}
60    * interface.</p>
61    *
62    * <p>For example, to create a secondary database that supports duplicates:</p>
63    *
64    * <pre>
65    * Database primaryDb; // The primary database must already be open.
66    * SecondaryKeyCreator keyCreator; // Your key creator implementation.
67    * SecondaryConfig secConfig = new SecondaryConfig();
68    * secConfig.setAllowCreate(true);
69    * secConfig.setSortedDuplicates(true);
70    * secConfig.setKeyCreator(keyCreator);
71    * SecondaryDatabase newDb = env.openSecondaryDatabase(transaction,
72    * "myDatabaseName",
73    * primaryDb,
74    * secConfig)
75    * </pre>
76    *
77    * <p>If a primary database is to be associated with one or more secondary
78    * databases, it may not be configured for duplicates.</p>
79    *
80    * <p><b>WARNING:</b> The associations between primary and secondary databases
81    * are not stored persistently. Whenever a primary database is opened for
82    * write access by the application, the appropriate associated secondary
83    * databases should also be opened by the application. This is necessary to
84    * ensure data integrity when changes are made to the primary database. If the
85    * secondary database is not opened, it will not be updated when the primary is
86    * updated, and the references between the databases will become invalid.
87    * (Note that this warning does not apply when using the {@link
88    * com.sleepycat.persist DPL}, which does store secondary relationships
89    * persistently.)</p>
90    *
91    * <h3><a name="transactions">Special considerations for using Secondary
92    * Databases with and without Transactions</a></h3>
93    *
94    * <p>Normally, during a primary database write operation (insert, update or
95    * delete), all associated secondary databases are also updated. However, when
96    * an exception occurs during the write operation, the updates may be
97    * incomplete. If the databases are transactional, this is handled by aborting
98    * the transaction to undo the incomplete operation. If an auto-commit
99    * transaction is used (null is passed for the transaction), the transaction
100    * will be aborted automatically. If an explicit transaction is used, it
101    * must be aborted by the application caller after the exception is caught.</p>
102    *
103    * <p>However, if the databases are non-transactional, integrity problems can
104    * result when an exception occurs during the write operation. Because the
105    * write operation is not made atomic by a transaction, references between the
106    * databases will become invalid if the operation is incomplete. This results
107    * in a {@link SecondaryIntegrityException} when attempting to access the
108    * databases later.</p>
109    *
110    * <p>A secondary integrity problem is persistent; it cannot be resolved by
111    * reopening the databases or the environment. The only way to resolve the
112    * problem is to restore the environment from a valid backup, or, if the
113    * integrity of the primary database is assumed, to remove and recreate all
114    * secondary databases.</p>
115    *
116    * <p>Therefore, secondary databases and indexes should always be used in
117    * conjunction with transactional databases and stores. Without transactions,
118    * it is the responsibility of the application to handle the results of the
119    * incomplete write operation or to take steps to prevent this situation from
120    * happening in the first place.</p>
121    *
122    * <p>The following exceptions may be thrown during a write operation, and may
123    * cause an integrity problem in the absence of transactions.</p>
124    * <ul>
125    * <li>{@link SecondaryConstraintException}, see its subclasses for more
126    * information.</li>
127    * <li>{@link LockConflictException}, when more than one thread is accessing
128    * the databases.</li>
129    * <li>{@link EnvironmentFailureException}, if an unexpected or system failure
130    * occurs.</li>
131    * <li>There is always the possibility of an {@link Error} or an unintended
132    * {@link RuntimeException}.</li>
133    * </ul>
134    */

522        * obsolete (SecondaryAssociation.getPrimary returns null), delete the
523        * record.
524        */
525       private boolean deleteObsoletePrimaryKeysInternal(final Cursor cursor,
526                                                         final Locker locker,
527                                                         final DatabaseEntry key,
528                                                         final DatabaseEntry data,
529                                                         final int batchSize) {
530           /* TODO: use dirty-read scan with mode to return deleted records. */
531           final LockMode scanMode = LockMode.RMW;
532           OperationResult searchResult;
533           if (key.getData() == null) {
534               /* Start at first key. */
535               searchResult = cursor.position(key, data, scanMode, null, true);
536           } else {
537               /* Resume at key/data pair last processed. */
538               searchResult = cursor.search(
539                   key, data, scanMode, null, SearchMode.BOTH_RANGE, false);
540               if (searchResult == null) {
541                   searchResult = cursor.search(

543               }
544           }

546           while (searchResult != null) {

548                   return true;

553               }
554               searchResult = cursor.retrieveNext(
555                   key, data, scanMode, null, GetMode.NEXT);
556           }
557           return false;
558       }

693        * Deletes the record associated with the given secondary key. In the
694        * presence of duplicate keys, all primary records associated with the
695        * given secondary key will be deleted.
696        *
697        * <p>When multiple primary records are deleted, the expiration time in the
698        * returned result is that of the last record deleted.</p>
699        *
700        * <p>When the primary records are deleted, their associated secondary
701        * records are deleted as if {@link Database#delete} were called. This
702        * includes, but is not limited to, the secondary record referenced by the
703        * given key.</p>
704        *
705        * @param key the key used as
706        * <a href="DatabaseEntry.html#inParam">input</a>.
707        *
708        * <!-- inherit other javadoc from overridden method -->
709        */
710       @Override
711       public OperationResult delete(final Transaction txn,
712                                     final DatabaseEntry key,
713                                     final WriteOptions options) {
714  
715           checkEnv();
716           final DatabaseImpl dbImpl = checkReadable();
717  
718           trace(Level.FINEST, "SecondaryDatabase.delete", txn, key, null, null);
719  
720           final CacheMode cacheMode =
721               options != null ? options.getCacheMode() : null;
722  
723           final Locker locker = LockerFactory.getWritableLocker(
724               envHandle,
725               txn,
726               dbImpl.isInternalDb(),
727               isTransactional(),
728               dbImpl.isReplicated()); // autoTxnIsReplicated
729  
730           OperationResult commitResult = null;
731           try {
732               final LockMode lockMode = locker.isSerializableIsolation() ?
733                   LockMode.RMW :
734                   LockMode.READ_UNCOMMITTED_ALL;
735  
736               try (Cursor cursor = new Cursor(this, locker, null)) {
737                   /* Read the primary key (the data of a secondary). */
738  
739                   final DatabaseEntry pKey = new DatabaseEntry();
740  
741                   OperationResult searchResult = cursor.search(
742                       key, pKey, lockMode, cacheMode, SearchMode.SET, false);
743  
744                   /*
745                    * For each duplicate secondary key, delete the primary record
746                    * and all its associated secondary records, including the one
747                    * referenced by this secondary cursor.
748                    */
749                   while (searchResult != null) {
750                       final Database primaryDb = getPrimary(pKey);
751  
752                       if (primaryDb == null) {
753  
754                           /* Primary was removed from the association. */
755                           cursor.deleteNoNotify(
756                               null, dbImpl.getRepContext());
757                       } else {
758  
759                           commitResult = primaryDb.deleteInternal(
760                               locker, pKey, cacheMode);
761  
762                           if (commitResult == null) {
763                               if (lockMode != LockMode.RMW) {
764  
765                                   /*
766                                    * The primary record was not found. The index
767                                    * may be either corrupt or the record was
768                                    * deleted between finding it in the secondary
769                                    * without locking and trying to delete it. If
770                                    * it was deleted or expired then just skip it.
771                                    */
772                                   if (cursor.checkCurrent(LockMode.RMW, null) !=
773                                       null) {
774  
775                                       /* There is a secondary index entry */
776                                       throw secondaryRefersToMissingPrimaryKey(
777                                           locker, key, pKey,
778                                           searchResult.getExpirationTime());
779                                   }
780                               } else {
781                                   if (!cursor.cursorImpl.isProbablyExpired()) {
782  
783                                       /* There is a secondary index entry. */
784                                       throw secondaryRefersToMissingPrimaryKey(
785                                           locker, key, pKey,
786                                           searchResult.getExpirationTime());
787                                   }
788                               }
789                           }
790                       }
791  
792                       checkOpen();
793  
794                       searchResult = cursor.retrieveNext(
795                           key, pKey, lockMode, null, GetMode.NEXT_DUP);
796                   }
797  
798                   if (commitResult == null) {
799                       dbImpl.getEnv().incDeleteFailOps(dbImpl);
800                   }
801  
802                   return commitResult;
803               }
804           } catch (Error E) {
805               envHandle.invalidate(E);
806               throw E;
807           } finally {
808               locker.operationEnd(commitResult != null);
809           }
810       }
