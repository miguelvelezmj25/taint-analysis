35  /**
36    * This class can generate the properties object that is normally loaded from
37    * the <code>GenericObjectEditor.props</code> file (= PROPERTY_FILE). It takes
38    * the <code>GenericPropertiesCreator.props</code> file as a template to
39    * determine all the derived classes by checking the classes in the given
40    * packages (a file with the same name in your home directory overrides the the
41    * one in the weka/gui directory/package). <br>
42    * E.g. if we want to have all the subclasses of the <code>Classifier</code>
43    * class then we specify the superclass ("weka.classifiers.Classifier") and the
44    * packages where to look for ("weka.classifiers.bayes" etc.):
45    * <p>
46    * <pre>
47    *
48    * weka.classifiers.Classifier=\
49    * weka.classifiers.bayes,\
50    * weka.classifiers.functions,\
51    * weka.classifiers.lazy,\
52    * weka.classifiers.meta,\
53    * weka.classifiers.trees,\
54    * weka.classifiers.rules
55    *
56    * </pre>
57    * <p>
58    * This creates the same list as stored in the
59    * <code>GenericObjectEditor.props</code> file, but it will also add additional
60    * classes, that are not listed in the static list (e.g. a newly developed
61    * Classifier), but still in the classpath. <br>
62    * <br>
63    * For discovering the subclasses the whole classpath is inspected, which means
64    * that you can have several parallel directories with the same package
65    * structure (e.g. a release directory and a developer directory with additional
66    * classes). <br>
67    * <br>
68    * The dynamic discovery can be turned off via the <code>UseDyanmic</code>
69    * property in the props file (values: true|false).
70    *
71    * @author FracPete (fracpete at waikato dot ac dot nz)
72    * @version $Revision: 13476 $
73    * @see #CREATOR_FILE
74    * @see #PROPERTY_FILE
75    * @see #USE_DYNAMIC
76    * @see GenericObjectEditor
77    * @see weka.core.ClassDiscovery
78    */
